## 集成结论

本次集成针对真机 `bbc478f4` 上的后台休息计时和震动停止问题。7 线程证据表明后台 Alarm 能触发，权限也正常；剩余问题集中在 App 逻辑：休息结束震动可能重复启动，进入下一组/结束训练没有完整清理所有提醒路径。1 线程已修复计时、WakeLock、单一停止入口和 PREPARE 占位问题；6 线程已补充训练页状态提示。

## 关键变化

- 版本升级到 `1.1.21 / versionCode 23`。
- 正式训练路径移除 `RestTimerService.prepare()` 占位通知，只有真实创建 `ActiveRestTimerEntity` 后才启动休息服务。
- `RestTimerService` 使用 partial wake lock 覆盖休息期，计时逻辑运行在独立 `Dispatchers.Default` scope。
- 休息结束通知切换到 `rest_finished_silent_v2`，避免旧通道振动设置残留。
- `RestTimerAlert` 增加重复震动保护，持续震动由统一入口启动和取消。
- `RestTimerReceiver` 优先启动 `RestTimerService.ACTION_FINISH`，只有服务启动失败时才兜底显示通知并震动。
- `TrainingViewModel.skipRest()` 和 `endWorkout()` 调用 `RestTimerController.stopRestAlert()`，统一清理 Alarm、通知、震动和服务。
- `MainActivity` 处理通知打开训练携带的停止提醒 intent。
- 训练页 UI 明确展示训练中、休息中、休息结束三类状态。
