# v1.1.22 Receiver 立即触发休息结束提醒热修复

## 背景

用户反馈 `v1.1.21` 发布包安装后，正式训练休息倒计时切后台仍可能不震动；但 `1` 线程单独调试时已经验证通过。因此本次按发布包回归处理。

## 检查结果

- `v1.1.21` APK 中已包含：
  - `WAKE_LOCK`
  - `USE_EXACT_ALARM`
  - `SCHEDULE_EXACT_ALARM`
  - `RestTimerReceiver`
  - `RestTimerService`
  - `versionName=1.1.21`
  - `versionCode=23`
- 当前 ADB 只检测到 `97257126520013`，这是禁止使用的 5G 移动终端；允许调试的 `bbc478f4` 不在线，因此未进行真机安装和抓日志。

## 根因判断

`v1.1.21` 仍依赖 `RestTimerReceiver -> startForegroundService(ACTION_FINISH) -> RestTimerService.startFinishedAlert()` 之后再启动持续震动。部分真机/OEM 后台策略下，即使 Alarm 到点，前台服务启动或进入 `startFinishedAlert()` 仍可能被延迟或吞掉，导致用户感知为“后台不震动”。

## 修复

修改文件：

- `packages/StreetStrengthApp-source-20260422/app/src/main/java/com/codex/streetstrength/timer/RestTimerReceiver.kt`
- `packages/StreetStrengthApp-source-20260422/app/build.gradle.kts`
- `README.md`

关键变化：

- Alarm 到点后，`RestTimerReceiver` 在确认 timer 应提醒后，立即执行：
  - `RestTimerAlert.showFinishedNotification(context)`
  - `RestTimerAlert.startContinuousVibration(context)`
- 随后仍启动 `RestTimerService.startFinishFromAlarm(context, timerId)` 让前台服务接管通知状态。
- 如果 Service 再次调用 `startContinuousVibration()`，`RestTimerAlert.vibrationActive` 会直接拦截，避免重启第二个持续震动 token。
